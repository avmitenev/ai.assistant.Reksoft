
## **Коммерсант: Для российского кода откроют хранилище**


#### О создании аналога GitHub с комментарием заместителя директора по инновационным рынкам «Рексофт», Игоря Кравченко


В Минэкономики предложили за 2,1 млрд руб. создать в стране аналог принадлежащего Microsoft сервиса GitHub — хранилища открытого кода. Это необходимо на случай отключения России от GitHub и других подобных мировых репозиториев из\-за санкций. Подобный сервис вряд ли будет пользоваться популярностью, но может быть полезен, если наполнять его кодом, разработанным за государственный счет, считают эксперты.


На создание российского аналога GitHub предложено выделить 2,1 млрд руб., сообщается в последней версии федерального проекта «Искусственный интеллект» (ИИ; есть у “Ъ”), разработанного Минэкономики и Сбербанком. В Минэкономики подтвердили “Ъ”, что идея российского аналога GitHub актуальна и является одним из «ключевых мероприятий для обеспечения условий для создания открытых библиотек ИИ» в рамках реализации национальной стратегии развития ИИ до 2030 года, утвержденной президентом РФ в октябре 2019 года. Проект еще не одобрен правительством.




***GitHub** — крупнейший сервис для хостинга и совместной разработки IT\-проектов, объединяющий хранилище (репозиторий) более 100 млн проектов с открытым кодом и социальную сеть для разработчиков. Услугами платформы пользуются более 40 млн разработчиков по всему миру и более 1,5 млн компаний, включая Apple, Amazon и Google. Летом 2018 года Microsoft приобрела GitHub за $7,5 млрд.*


Необходимость создания аналога GitHub объясняется риском отключения России от зарубежных хранилищ кода «из\-за репутационных и санкционных рисков». В российский репозиторий предлагается копировать наиболее востребованные программы из зарубежных открытых источников. Отечественный GitHub может появиться к 2021 году, а к 2024 году таких репозиториев должно стать пять, предполагает проект.


Риски ограничения доступа к GitHub, безусловно, существуют, считает **заместитель директора по инновационным рынкам «Рексофт» Игорь Кравченко**. Но полное закрытие доступа — задача непростая, отмечает он, приводя в пример ситуацию с Telegram. С учетом «минимальных затрат на реализацию» создание российского репозитория имеет смысл для подстраховки от внезапного отключения России от того или иного зарубежного сервиса, считает член правления ассоциации «Руссофт» Владимир Рубанов. «Однако наибольшую ценность подобный сервис приобретет, только если на нем самом станут возникать уникальные мастер\-репозитории новых проектов. Это станет возможно, если привлечь разработчиков из России и стран\-союзников, которые заинтересованы в создании технологически независимой от США площадки по совместной работе над открытыми проектами»,— рассуждает он.


Другие участники рынка относятся к идее скептически. «Российским аналогом никто не будет пользоваться, пока есть GitHub»,— уверен технический директор компании «Системы компьютерного зрения» (входит в ГК «Ланит») Михаил Смирнов. Он подчеркивает, что подобные сервисы мирового уровня живут за счет сообщества, а не административного ресурса. Важно сделать акцент не на копировании GitHub или аналогичных платформ, а на разработке конкурентоспособных ресурсов и библиотек, создании живого сообщества вокруг платформы, согласен гендиректор «ABBYY Россия» Дмитрий Шушкин.


*125 миллиардов рублей предлагается выделить на федеральный проект «Искусственный интеллект» до 2024 года*


Но на российском аналоге не соберешь большие сообщества, он будет «напоминать ситуацию, когда накрыли и украсили стол, а гости не пришли», считает заместитель директора по специальным программам ICL Services Андрей Крехов. GitHub стал популярен за счет идеи и возможностей для широкомасштабного социального взаимодействия разработчиков на достаточно мощном и бесплатном инструментарии, объясняет он. Сейчас для многих компаний GitHub также стал фактически HR\-платформой для поиска разработчиков и обучающей платформой, указывает господин Крехов. Пока власти не примут закон о запрете размещения исходных кодов российского софта на зарубежных серверах, «отдельный загончик для российских программистов никто создавать не будет», резюмирует он.


Польза от собственной копии GitHub может быть только в случае обязательной публикации в нем результатов разработки софта за бюджетные деньги, причем под открытой лицензией, считает гендиректор «Базальт СПО» Алексей Смирнов. С этим согласен исполнительный директор ассоциации «Отечественный софт» Ренат Лашин. Он добавляет, что такая площадка позволила бы значительно сократить расходы на поддержку и развитие софта, а также разработку новых программ за счет возможности повторного использования кода. Попыткой создать национальный репозиторий был Национальный фонд алгоритмов и программ, запущенный в 2014 году по заказу Минкомсвязи, напоминает Алексей Смирнов, но этот проект «был провальным».
